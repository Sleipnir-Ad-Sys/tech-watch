Technologies suivies

Cette plateforme surveille automatiquement les releases, articles et packages de ces projets. Le pipeline s'exécute chaque jour à 07h00, collecte les données et calcule un score d'impact.

⚙️ Comment fonctionne la veille ?

📡
1. Collecte

Chaque jour : API GitHub, flux RSS des blogs officiels, registre PyPI. Les données brutes sont normalisées en objets structurés.

⚡
2. Pipeline Polars

Transformation vectorisée avec Polars : parsing semver, déduplication, extraction des breaking changes (regex), scoring.

🦆
3. Stockage DuckDB

Chaque release est upsertée dans DuckDB (base analytique locale) et exportée en Parquet. Le delta est calculé à chaque run.

📊
4. Score d'impact

Score = (majeur×10 + mineur×3 + patch×0.5 + rupture×8 + fréquence×2) × poids_projet. Le RuleEngine applique des règles métier supplémentaires.

🎯 Niveaux d'impact

🔴 Critique Score ≥ 20

Version majeure avec rupture de compatibilité, ou faille de sécurité (CVE/RCE). Revue immédiate recommandée avant mise à jour.

Déclencheurs :
  • Version X.0.0 + BREAKING CHANGE
  • Mot-clé sécurité (CVE, RCE, vulnérabilité...)
  • Score calculé ≥ 20
🟠 Élevé Score ≥ 12

Version majeure sans breaking change explicite, ou version mineure avec changements structurels. Planifier la mise à jour.

Déclencheurs :
  • Version X.0.0
  • BREAKING CHANGE sans version majeure
  • Score calculé ≥ 12
🟡 Moyen Score ≥ 5

Version mineure avec nouvelles fonctionnalités ou améliorations. Pas d'urgence, à intégrer dans la prochaine itération.

Déclencheurs :
  • Version X.Y.0 (mineur significatif)
  • Fréquence de releases élevée
  • Score calculé ≥ 5
🟢 Faible Score > 0

Patch ou correction mineure. Mise à jour de routine, aucun risque de régression.

Déclencheurs :
  • Version X.Y.Z (patch seul)
  • Corrections de bugs documentées

DataFrame library ultra-rapide écrite en Rust. API lazy, multi-threadée, compatible Arrow. Alternative moderne à pandas avec des performances 10-50× supérieures.

Python · Rust PyPI pola.rs ↗
Pourquoi ? Utilisé comme moteur ETL de ce pipeline.

DuckDB
duckdb/duckdb

Base de données analytique embarquée (OLAP). Lit directement Parquet/CSV/JSON, compatible Arrow, SQL complet. Aucun serveur requis.

C++ · Python · SQL PyPI duckdb.org ↗
Pourquoi ? Entrepôt local de ce pipeline. Remplace un serveur PostgreSQL entier.

Apache Arrow
apache/arrow

Format de données en mémoire columnar, standard inter-langages. Zéro-copie entre Python, Rust, Java. Colle Polars ↔ DuckDB.

Multi-langages PyPI arrow.apache.org ↗
Pourquoi ? Pont zéro-copie entre Polars et DuckDB dans le pipeline.

API DataFrame backend-agnostique : même code pour DuckDB, BigQuery, Snowflake, Spark. Abstraction universelle pour la data analytique.

🚀 Guide de Migration

Pandas → Polars

Polars est 10–15× plus rapide que Pandas sur les opérations groupby/join et consomme ~60 % moins de mémoire grâce à son backend Apache Arrow et son exécution multi-threadée lazy. La migration est progressive — les deux peuvent coexister.

GroupBy : 12× Join : 8× Filter : 6× Mémoire : −60 % Multi-thread natif
Pandas
import pandas as pd

df = pd.read_csv("data.csv")

# GroupBy
result = (df
  .groupby("category")["value"]
  .mean()
  .reset_index()
)

# Filtre + sélection
out = df[df["score"] > 0.5][["id","score"]]

# Join
merged = df.merge(ref, on="key", how="left")
→
Polars
import polars as pl

df = pl.read_csv("data.csv")

# GroupBy (lazy → collect)
result = (df.lazy()
  .group_by("category")
  .agg(pl.col("value").mean())
  .collect()
)

# Filtre + sélection
out = df.filter(pl.col("score") > 0.5).select("id","score")

# Join
merged = df.join(ref, on="key", how="left")
💡 Lecture seule : pd.DataFrame → pl.DataFrame — pas de inplace=
💡 Index : Polars n'a pas d'index — utilisez une colonne explicite
💡 apply() : Remplacer par pl.Expr ou map_elements() pour garder la vitesse
💡 NaN vs null : Polars distingue null (missing) de NaN (float invalid)
